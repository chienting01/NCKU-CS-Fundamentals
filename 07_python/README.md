# Python

## 這一區在做什麼

這個資料夾用來系統化整理 Python 學習內容，重點不是只會寫幾行語法，而是逐步建立：

1. Python 基礎語法能力
2. 基本程式思維
3. 常用容器型別操作能力
4. 檔案與路徑處理能力
5. NumPy / Pandas 入門能力
6. 進入 Kaggle 前的前置準備能力

這一區的定位不是一般的語法抄寫區，而是**資料分析 / Kaggle 導向的 Python 學習模組**。

---

## 為什麼要學 Python

在這個專案中，Python 的角色和 C 不一樣。

- **C 語言**：偏底層、偏基礎能力、偏計算機科學核心訓練
- **Python**：偏工具性、偏資料處理、偏分析與 Kaggle 實作

學 Python 的目的不只是「會寫 Python」，而是為了之後能做：

- 讀取與處理 CSV
- 基本資料清理
- NumPy 數值運算
- Pandas 表格分析
- Kaggle 入門競賽實作

所以這一區會特別重視：

- 變數
- 流程控制
- 函數
- Python 容器型別
- 例外處理
- 檔案與路徑
- NumPy
- Pandas

## 結構說明

### `notes/`
放理論筆記、概念整理、主題說明。  
這裡主要回答的是：

- 這個主題是什麼
- 為什麼要學
- 核心概念是什麼
- 有哪些容易混淆的地方

### `basics/`
放單一主題的小型可執行範例。  
每個主題資料夾都應該包含：

- `main.py` 或該主題主要程式檔
- `README.md`

這裡主要回答的是：

- 這個小範例在練什麼
- 怎麼執行
- 會看到什麼結果

### `scripts/`
放比較整合型的 Python 腳本。  
通常用來練習：

- NumPy
- Pandas
- 小型資料處理流程

它比 `basics/` 更接近「功能腳本」或「整合練習」。

### `exercises/`
放練習題、練習解法、錯誤紀錄與重做紀錄。  
這裡不是單純存放題目，而是用來追蹤：

- 題目
- 解法
- 錯誤
- 修正
- 重做心得

## 學習原則

這一區採用以下原則：

1. 不只看，要自己寫
2. 每個主題至少有一個可執行 `.py` 檔
3. 每個實作資料夾都要有 `README.md`
4. 每個主要主題都要有一份中文 Markdown 筆記
5. 先學 Python 本體，再進資料分析工具
6. 學 Python 的目的是支援後續 Kaggle 與資料分析

## note 和 README 的分工

### `notes`
`notes` 是**知識筆記**。  
主要用來整理：

- 概念
- 原理
- 語法意義
- 常見錯誤
- 主題之間的關係

### `README`
`README` 是**資料夾或實作說明**。  
主要用來整理：

- 這個範例在做什麼
- 這個資料夾放什麼
- 怎麼執行
- 預期輸出是什麼

一句話區分：

- `note` 講的是**知識本身**
- `README` 講的是**這個實作怎麼看、怎麼跑**
